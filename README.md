## Мок-сервер
### Принцип работы

При старте сканируется папка env.

В ней на первом уровне находятся подпапки группирующие апи по порту и пути. Эта информация указывается в файле config.ts.

Далее сканируется папка routes каждой группы.

В ней каждая последующая подпапка является частью пути апи. Конечной частью пути апи является файл с функциями, которые описывают методы HTTP (GET, POST и т.д.). Метод DELETE соответствует функции с названием del.

### Сценарии мок-сервера

хранение: localStorage

структура ключа: method-<путь-апи>, где путь-апи: pathname

на сервере ожидается заголовок с именем x-scenario

пример, pathname === 'api/v1/module1/list', ключ localStorage GET-api-v1-module1-list, значение ключа http400, значение заголовка http400
